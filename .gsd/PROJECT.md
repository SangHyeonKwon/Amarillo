# PROJECT — Transaction Failure Intelligence

## 한 줄 정의

이미 가진 인덱서/디코더/trace 파이프라인을 살려, **"이 이더리움 트랜잭션이 왜 revert 됐는가"** 를
개별 진단 + 실시간 + 임베드 가능한 API로 제공한다.

## 왜 (엣지 근거)

Dune은 호스팅된 SQL 분석 도구다. 우리가 Dune을 이기려는 게 아니라, **Dune이 구조적으로 못 하는
영역**만 공략한다.

- **Dune이 전혀 안 하는 것 (`edge: untapped`)** — 개별 tx 실패 진단 API, 실시간 실패 스트림,
  실패 패턴 웹훅/액션, 비공개 데이터 조인
- **Dune이 약한 것 (`edge: weak-spot`)** — revert 사유 디코딩 + 의미 분류, 임베드용 실패 분석 API,
  저지연 실패 시계열
- **Dune이 강한 것 (= 버림)** — 일반 볼륨/풀/트레이더/가격 차트. 엣지 0. 추가 금지.

## 목적 우선순위

1. **학습** — 빌드 자체가 백엔드 커리큘럼. 각 슬라이스 = 개념 학습 + 구현.
2. **포트폴리오** — 기본 충족 상태. "RPC→디코드→trace→실시간 인덱서를 직접 설계"가 셀링포인트.
3. **제품** — 위 두 개를 충족하며 실수요(댑 개발자·지갑·봇 운영자·CS) 향해 전진.

## 범위 (스코프)

- 체인: 이더리움 메인넷 / 프로토콜: Uniswap V3 (기존 범위 유지, 확장 금지 — 깊이 우선)
- 기존 Rust 워크스페이스(`crates/{indexer,decoder,db,api}`) 위에 **추가**로 빌드. 재작성 금지.
- 프론트엔드(`web/`)는 **후순위** — API가 1급 산출물. 프론트는 소비자 1개일 뿐.

## 비범위 (안 함)

- 새 체인/프로토콜 확장, 멀티체인
- 일반 DeFi 분석 대시보드 (Dune 영역)
- gsd-2 CLI/런타임 DB 자체 구현 — 본 `.gsd/` 문서는 계획 산출물로만 사용
